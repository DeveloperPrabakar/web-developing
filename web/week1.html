Anatomy Of HTML Tag:

  • Usally HTML tag will have a open and end tag
  • Inbetween we can type the content
  • <p>--Content--</p>
  • Opening Tag Closing Tag (where 'P' is an element)
  • But<BR>and<HR>tags only has "Opening tag"
  • No closing tag at all
  • <BR>stands for "Line Break"
  • <HR>stands for "Horizontal rule"
  • Every HTML element can have predefined attributes
  • Each attribute has its "Own Value & meaning"
  • Attribute should be specified only in OPENING TAG
  • Example:<P ID = "Attribute Value">....Content.....</P>
  • Not necessary to provide the attribute value in single (Or) double quotes. However, that is a good practise

Basic HTML Document Structure:

  • Every HTML document should start with<!doctype html>either in upper case. or lower case letter.
  • <!doctype html> is just an indication to the broswer says "Webpage follows the standard".

HTML Contenet Models:

  • The term content model refers to the full behaviour of the browser, applies to the elements that belonging to the content model into the nesting rule of those elements.
  • Which elements has to be nested with other elements.

  • Traditonal Content Models

  • Block Level Elements
  • Render to begin on a newline (by default)
  • May contain inline or other block-level elements.
  • Inline Elements
  • Render on the same line (by default)
  • May contain other inline elements.
  • HTML 5 replaces these definitons with more complex set of content categories.
  • However, this disctinction remains practical becuase it aligns well with existing css rules.

Heading Elements&Semantic Elements:

  • Se/man/tic - Relating to meaning in language or logic.
  • Element that implies some menaing to the content. (Giving hints about the content)
  • Human / machine can understand meaning of content surrounded by a semantic element better.
  • May help search engine ranking, i.e., SEO
